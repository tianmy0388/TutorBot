# 隐私、内容安全与防幻觉说明

## 1. 数据分类

- 账户与会话：用户标识、会话标识、消息。
- 学习画像：知识掌握、认知风格、错误模式、节奏、动机和模态偏好。
- 学习事件：资源访问、练习结果、耗时和反馈。
- 系统密钥：LLM、Embedding 和搜索服务凭据。

比赛演示只使用合成数据。真实部署应提供用途说明、访问控制、导出和删除机制。

## 2. 最小必要原则

只收集完成个性化学习所需数据，不采集身份证号、精确位置等无关信息。日志使用用户内部标识，密钥仅保存在本地环境配置并以掩码回显。

## 3. 防幻觉流程

资源生成后依次执行声明抽取、知识库或外部证据核对、质量审核和安全检查。已验证事实展示引用；证据不足的内容进入 `unverified_claims`；被反驳或不安全的资源不得作为正常结果发布。

## 4. 攻击面与控制

| 风险 | 控制措施 |
|---|---|
| 提示注入 | 限定检索范围，区分系统指令与文档内容，不执行文档中的工具指令 |
| 恶意文件 | 类型与大小检查，解析失败进入明确终态 |
| 代码执行 | 临时目录、超时、受限解释器、产物白名单和错误合同 |
| 路径穿越 | 资源下载接口校验用户、资源 ID 和解析后路径 |
| 密钥泄露 | `.env` 忽略、掩码回显、提交前 secret scan |
| 错误学术内容 | 引用、逐条事实判断、未验证声明提示和人工复核 |

## 5. 发布前安全测试

至少覆盖提示注入、越权资源读取、路径穿越、恶意代码、敏感内容、伪造引用和密钥回显。测试报告记录输入、预期、实际结果和修复状态。

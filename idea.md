&#x20; 二、核心 Agent 角色与职责



&#x20; 2.1 画像构建 Agent 集群



&#x20; ┌────────────────┬────────────────────────────┬────────────────────────────────────────────────────────┐

&#x20; │     Agent      │            职责            │                        关键实现                        │

&#x20; ├────────────────┼────────────────────────────┼────────────────────────────────────────────────────────┤

&#x20; │ 特征抽取 Agent │ 从对话中提取结构化特征     │ LLM + JSON Schema 约束输出，类似 DT 的 ConceptAnalysis │

&#x20; ├────────────────┼────────────────────────────┼────────────────────────────────────────────────────────┤

&#x20; │ 认知诊断 Agent │ 通过对话式探测评估知识掌握 │ 基于 IRT/DKT 的轻量模型 + LLM 对话                     │

&#x20; ├────────────────┼────────────────────────────┼────────────────────────────────────────────────────────┤

&#x20; │ 画像更新 Agent │ 随学随新，增量更新画像     │ 差异检测：比较新旧特征，只更新变化部分                 │

&#x20; └────────────────┴────────────────────────────┴────────────────────────────────────────────────────────┘



&#x20; 六维画像数据模型：

&#x20; class LearnerProfile(BaseModel):

&#x20;     # 维度1: 知识基础

&#x20;     knowledge\_map: dict\[str, float]  # { "反向传播": 0.8, "RNN": 0.2, "LSTM": 0.0 }



&#x20;     # 维度2: 认知风格

&#x20;     cognitive\_style: str  # "visual" | "verbal" | "deductive" | "inductive" | "active" | "reflective"



&#x20;     # 维度3: 易错点偏好

&#x20;     error\_patterns: list\[ErrorPattern]  # \[{concept, mistake\_type, frequency}]



&#x20;     # 维度4: 学习节奏

&#x20;     learning\_pace: PaceProfile  # {avg\_session\_duration, preferred\_chunk\_size, review\_interval}



&#x20;     # 维度5: 动机与目标

&#x20;     motivation\_profile: MotivationProfile  # {goal\_type, urgency, self\_efficacy\_score}



&#x20;     # 维度6: 模态偏好

&#x20;     modality\_preferences: dict\[str, float]  # {"text": 0.6, "video": 0.8, "interactive": 0.7, "diagram": 0.9}



&#x20; 画像构建流程（借鉴 DT 的对话流）：

&#x20; 用户: "我是CS研一，想学LSTM，之前学过基础NN但对RNN不太熟"

&#x20;         ↓

&#x20; 特征抽取Agent  → 提取: {major: CS, level: grad, knows(NN): true, knows(RNN): low, goal: LSTM}

&#x20;         ↓

&#x20; 认知诊断Agent → 追问2-3个探测问题（如"你能描述一下反向传播的基本思想吗？"）

&#x20;         ↓

&#x20; 画像初始化    → 生成初始6维画像

&#x20;         ↓

&#x20; 存入画像Store → 后续每次交互自动触发画像更新Agent

&#x20; 路径规划流程：

&#x20; 1. 定位：根据画像的 knowledge\_map 在知识图谱中定位学生当前位置

&#x20; 2. 剪枝：已掌握节点标记为"可跳过"，生成学习子图

&#x20; 3. 排序：拓扑排序 + 学习依赖 → 推荐学习顺序

&#x20; 4. 匹配：为每个节点匹配已生成的/可生成的资源

&#x20; 5. 推送：按顺序 + 画像的 learning\_pace 控制推送节奏



&#x20; 知识图谱存储：Neo4j 或直接用 JSON/YAML 文件定义（轻量方案），结构参考：

&#x20; nodes:

&#x20;   - id: lstm

&#x20;     name: "长短期记忆网络 (LSTM)"

&#x20;     category: deep\_learning

&#x20;     difficulty: 3

&#x20;     prerequisites: \[rnn, backpropagation]

&#x20;     estimated\_hours: 8

&#x20;     resources: \[]  # 动态填充

&#x20; 四、关键数据结构设计



&#x20; # 资源包

&#x20; class ResourcePackage(BaseModel):

&#x20;     package\_id: str

&#x20;     topic: str

&#x20;     created\_at: datetime

&#x20;     resources: list\[Resource]

&#x20;     learning\_path: LearningPath

&#x20;     target\_profile\_snapshot: LearnerProfile  # 生成时的画像快照



&#x20; # 单个资源

&#x20; class Resource(BaseModel):

&#x20;     resource\_id: str

&#x20;     type: Literal\["document", "mindmap", "exercise", "reading", "video", "code"]

&#x20;     title: str

&#x20;     content: str  # Markdown 正文

&#x20;     format\_specific: dict  # 类型特定字段

&#x20;     # document: {sections: \[...]}

&#x20;     # mindmap: {mermaid\_dsl: "..."}

&#x20;     # exercise: {questions: \[{type, question, options, answer, explanation}]}

&#x20;     # code: {language, code, explanation, sandbox\_url}

&#x20;     # video: {manim\_code, video\_url, thumbnail\_url}

&#x20;     difficulty: int  # 1-5

&#x20;     estimated\_minutes: int

&#x20;     prerequisites: list\[str]  # 前置知识点

&#x20;     generated\_by: list\[str]  # 参与的Agent列表



&#x20; # 学习路径

&#x20; class LearningPath(BaseModel):

&#x20;     nodes: list\[PathNode]

&#x20;     edges: list\[tuple\[str, str]]  # from\_node\_id → to\_node\_id



&#x20; class PathNode(BaseModel):

&#x20;     node\_id: str

&#x20;     concept: str

&#x20;     resources: list\[str]  # resource\_ids

&#x20;     status: Literal\["locked", "available", "in\_progress", "completed"]

&#x20;     required\_for: list\[str]  # 解锁哪些后续节点
